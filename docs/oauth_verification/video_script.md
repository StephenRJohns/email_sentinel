# OAuth Review Demo Video Script — emAIl Sentinel™

## Length: ~3–5 minutes

### 1. Introduction (10–15 sec)
“emAIl Sentinel is a Gmail add-on that monitors labels selected by the user and sends alerts based on user-defined rules. It runs entirely inside the user’s Google account.”

---

### 2. Installation (20 sec)
- Show installing from Workspace Marketplace
- Open Gmail and launch the add-on

---

### 3. Configuration (45 sec)
- Show settings panel
- Select a Gmail label (e.g., “Invoices”)
- Create a simple rule (e.g., “If subject contains invoice”)

**Say:**  
“The app only accesses messages in labels explicitly selected by the user.”

---

### 4. Data Access Demonstration (45 sec)
- Open a matching email
- Trigger evaluation

**Say:**  
“We read message metadata and a body excerpt only to evaluate the rule and generate an alert. Data is not used for advertising or stored externally.”

---

### 5. Output Channels (45 sec)
- Enable Calendar or Sheets
- Show event/row created

**Say:**  
“Data is sent only to integrations the user enables.”

---

### 6. No Backend Architecture (20 sec)
**Say:**  
“The app runs in Google Apps Script within the user’s account. We do not operate a backend database.”

---

### 7. Wrap-Up (10 sec)
**Say:**  
“User data is used solely to provide the requested alerting functionality.”

---

## Tips
- Use a clean test account
- Show real UI (not mockups)
- Keep narration simple and factual
