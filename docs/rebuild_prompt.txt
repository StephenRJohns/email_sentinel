=== WHAT THIS PROJECT IS ===

emAIl Sentinel (exact branding: lowercase e, lowercase m, uppercase A, uppercase I, lowercase l, space, uppercase S, lowercase entinel) is a production Google Workspace Gmail Add-on built with Google Apps Script (V8 runtime). It watches Gmail labels for new messages, evaluates each one against user-defined plain-English rules using the Google Gemini REST API, and dispatches alerts via SMS, Google Chat, Google Calendar, Google Sheets, Google Tasks, or MCP server endpoints.

Operated by JJJJJ Enterprises, LLC (jjjjj_enterprises_llc@protonmail.com).

=== TECH STACK ===

- Runtime: Google Apps Script (server-side JavaScript, V8 runtime)
- Host: Google's infrastructure — no external backend, no database
- UI: Google Cards Service (Gmail sidebar add-on cards)
- State: PropertiesService.getUserProperties() — per-user, per-script, private, 9 KB per value limit
- AI: Google Gemini REST API (UrlFetchApp to generativelanguage.googleapis.com)
- Auth: OAuth 2.0 scopes declared in appsscript.json; ScriptApp.getOAuthToken() for Google-native channels
- No npm, no build step, no transpilation. Deploy with `clasp push`.

=== DATA MODEL ===

All state in PropertiesService.getUserProperties():

  mailsentinel.settings — JSON: Gemini key/model, poll interval, business hours, SMS config, alert channel IDs, license tier
  mailsentinel.rules    — JSON array of rule objects (see RulesManager.gs header for schema)
  mailsentinel.seen     — JSON: per-label array of recently-seen Gmail message IDs (dedup baseline)
  mailsentinel.log      — Ring buffer of last ~60 activity log lines

=== ALERT CHANNELS ===

SMS:             BYO provider — six quick-start presets (Textbelt, Telnyx, Plivo, Twilio, ClickSend, Vonage)
                 plus Generic webhook (POST {"to": "+1...", "body": "..."} to any HTTPS endpoint)
Google Chat:     Webhook POST to user-configured Chat Space URLs (Pro only)
Google Calendar: Creates 15-min event via CalendarApp (uses 'calendar' OAuth scope)
Google Sheets:   Appends row via SpreadsheetApp (uses 'spreadsheets' OAuth scope)
Google Tasks:    Creates task via Tasks REST API + ScriptApp.getOAuthToken() (uses 'tasks' scope)
External integrations: Settings section labeled "External integrations" covers MCP servers (JSON-RPC 2.0 tools/call over HTTPS) plus the Asana REST API direct path. Type dropdown options: Custom (default for new servers, lead recommendation — Help has a Cloudflare Worker walkthrough that gets you a working endpoint in 15 min), Microsoft Teams (Microsoft Graph MCP, send_message tool, OAuth + admin consent friction), Asana (REST API — easier, PAT-based, not strictly MCP), Asana (MCP V2 — requires OAuth, asana_create_task tool, hourly token expiry). Slack was removed from the dropdown because Slack does not host an MCP server; Slack users self-host an MCP-to-Slack bridge and pick Custom. Microsoft 365 was renamed to Microsoft Teams since Teams chat is the actual alert surface. Legacy stored types `slack` and `ms365` are migrated at editor render time (slack→custom, ms365→teams) so existing user configs survive the rename.

=== KEY DESIGN CONSTRAINTS ===

- Apps Script execution limit: 6 min. runMailCheck() guards with MAX_RUN_MS=180000 (3 min), MAX_EVALS_PER_RUN=100
- 9 KB UserProperties limit per key. saveRules() throws if over. saveSeen_() auto-shrinks buffer
- Log buffering: startLogBuffering() + flushLog() batch writes during trigger runs
- Concurrency guard: LockService.getUserLock(0ms timeout) in runMailCheck(); skip if locked
- Cards are fully stateless: read UserProperties fresh on every render
- In-card Home button on the four root cards (Rules, Settings, Activity log, Help): unconditionally prepended via homeButtonSection_(). Wired to handleGoHome which does popToRoot + updateCard(buildHomeCard()). Always-on rather than conditional because Apps Script doesn't expose navigation-stack depth at handler time, so updateCard refreshes (rule toggle, log refresh, settings save) would silently drop a conditional Home button. Mild redundancy with Gmail's back arrow on push paths is accepted. Starter rules card is the one nav target without it — only reached via push from home, back arrow always available.
- Polling grid: Gmail / Workspace add-on time-based triggers must fire >= 60 min apart. enforcePollFloor() snaps pollMinutes up to the nearest multiple of 60 (Free and Pro both have minPollMinutes=60). installTrigger() uses everyHours(targetHours) where targetHours = round(pollMinutes/60). runMailCheck() still applies a LAST_RUN_KEY elapsed-time skip-check for precise cadence when the user picks longer-than-1-hour intervals. runMailCheck({force: true}) bypasses the skip-check for manual "Scan email now" — the way users get sub-hour responsiveness on demand.
- Home polling dropdown auto-saves: the home card "Scan email every" SelectionInput wires setOnChangeAction → handleHomePollChange, which persists the picked value to settings.pollMinutes the moment the user changes the selection (CardService doesn't auto-submit form inputs on navigation; without this the user could pick a value, navigate to Settings, and find the old value still selected). handleHomePollChange returns an empty ActionResponseBuilder so the save is silent — no toast, no card refresh. handleStartMonitoring re-saves the form value as defense-in-depth.
- Action button color conventions (Cards.gs): BRAND_PURPLE_ #581c87 = primary FILLED CTAs (Save, Generate, + New rule, Start scheduled scans); BRAND_PURPLE_LIGHT_ #7c3aed = secondary FILLED CTA when a primary purple sits beside it (Scan email now); BRAND_RED_ #c62828 = every Delete button (rule list + 3 editor cards + their 4 confirmation cards = 8 total) with whiteText_ wrapper; BRAND_YELLOW_LIGHT_ #fde68a = rule toggle when the rule is currently enabled (button reads "Disable") with blackText_ wrapper. When the rule toggle reads "Enable" (rule is OFF) it stays plain text — enabling isn't cautionary. Text color must be wrapped explicitly via whiteText_ / blackText_ since CardService doesn't auto-pick contrasting text reliably for FILLED buttons with custom backgrounds.
- Phone number entry: country-code dropdown (~45 countries with flag emoji + dial code, default +1) plus digits-only input. splitPhoneNumber_/combinePhoneNumber_ handle E.164 storage and round-trip on edit.
- AI assist gating: "Help me write the rule text" is Pro-only; "Help me write the alert text" is available on Free. Both buttons open a description-input card; the user types informally, Gemini cleans it into a rule description or alert format. The alert-text flow is channel-aware — it factors in whichever channels are ticked on the rule.
- First-run baseline: first check per label records seen IDs, no alerts (intentional behavior)
- Gemini called via REST (UrlFetchApp), not SDK — only needs script.external_request scope
- Email body: only first 2,000 characters sent to Gemini
