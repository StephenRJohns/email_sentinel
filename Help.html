MailAlert watches your Gmail for new messages and sends an alert when one matches a rule you describe in plain English. Rules are evaluated by Google Gemini.

Quick start
1. Open Settings and paste your Gemini API key. Get one free at aistudio.google.com/app/apikey.
2. (Optional) Configure SMS — pick Twilio or a generic webhook provider and fill in the credentials.
3. Open Rules and click + New rule.
4. Back on the home card, click Start monitoring. A time-driven trigger will run in the background even when Gmail is closed.

Writing a rule
Rules are plain English. Be specific about senders, subjects, attachments, or body keywords. Examples:
• "Any email from anyone @tma.com that has a PDF attachment that looks like an invoice or purchase order."
• "Subject contains URGENT or CRITICAL."
• "Email from boss@company.com asking for a status update."

Each rule has an Alert message format field — plain-English instructions that tell Gemini how to format the alert that recipients receive. The default includes date, sender, subject, summary, and any action items.

Labels (vs. folders)
Gmail uses labels rather than folders. Use INBOX for the inbox, or any label name as it appears in Gmail. Labels containing spaces are fine — type them exactly as shown. Multiple labels: comma-separated.

Alert channels
Email — sent from your own Gmail account. No SMTP server required.
SMS — Google Workspace doesn't ship an SMS API, so MailAlert delegates to your provider. Twilio works out of the box (paste Account SID, Auth Token, and "From" number in Settings). For other providers (MessageBird, Vonage, AWS SNS, in-house gateway, …) pick the Generic webhook provider and point it at any HTTPS endpoint that accepts a JSON body of {"to":"+15551234567","body":"…"}.

Business hours
Optionally restrict checks to a daily window. Outside of business hours the trigger still fires but skips the actual mail check, so you don't burn Gemini quota overnight.

Polling
Apps Script time-driven triggers fire every 1, 5, 10, 15, or 30 minutes. The first run for each label baselines existing messages so you don't get an alert flood when you first install — alerts start with the next new message.

Privacy
Settings, rules, the seen-message baseline, and the activity log are all stored in UserProperties, which is private to your own Google account. Email contents are sent only to Google's Gemini API and to whatever SMS provider you configure — never anywhere else.

Troubleshooting
• "No Gemini API key configured" — open Settings and paste a key, then click Test Gemini.
• "Label '…' fetch failed" — verify the label exists in Gmail with that exact name (case-insensitive).
• SMS not delivered — open the Activity Log to see the provider's error response.
• Alerts firing for old mail after install — open Settings and click Reset baseline; the next run will re-baseline.
